# 初赛轮

初赛阶段主要考察选手对于向量、向量索引的了解程度，允许在不修改源代码的基础上进行调优，通过调整向量索引构建与查询的参数，获取最佳的向量查询性能。

祝开发者们快速融入，享受竞赛乐趣。

## 测评环境

在提交后数据库将被部署在一个 16C 32GB Intel Xeon(Emerald Rapids) Platinum 8575C 的容器环境中进行测试，数据库运行镜像基于 [polardb-pg-docker-devel-ubuntu24.04](https://github.com/ApsaraDB/polardb-pg-docker-images/blob/main/Dockerfile-devel-ubuntu24.04)

## 测评流程

测评分为三个阶段：

*   **数据库初始化阶段**：预计约 5 分钟，测评程序将运行回归测试验证数据库、pgvector 插件基础功能的正确性，如果回归测试通过则会完成数据库、测试账号以及基础表结构的创建。
    
*   **数据导入与索引构建阶段**：限时 60 分钟，测评程序会将向量数据导入到数据库中，并建立向量索引。因选手提供的向量构建参数不同，索引构建的时间也不尽相同，选手需要将索引构建时间控制在指定区间内
    
*   **基准测试阶段**：根据选手提供的向量查询参数进行多轮查询，测评系统将返回召回率（Recall）超过 0.85 时的最大每秒查询数（Query Per Second，QPS），作为选手的成绩
    

初赛与复赛的差异:

*   初赛不将索引构建的时间纳入成绩，向量与索引数据使用全内存方式存储，不存在 IO 过程
    
*   因测评资源限制，在线测评运行时间较短
    

## 注意事项

*   初赛主要考察选手对于向量操作与向量索引的熟悉程度，不要求选手修改代码，不过我们也鼓励选手通过修改源代码的方式获得更好的成绩
    
*   由于测评资源的限制，不建议选手使用太大的索引构建与查询参数
